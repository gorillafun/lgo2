# 第2章: 基本型と宣言

この章では、Go言語における基本的なデータ型（整数、浮動小数点数、文字列、ブール値など）と、変数や定数の宣言方法について学びます。
プログラムがデータをどのように扱い、メモリ上に保存するかを理解するための重要なステップです。

## 詳細解説

### 1. 文法の実験場 (`ex0200.go`)

このファイルは、第2章で登場する様々な文法事項をまとめて試すためのコードです。
非常に長いですが、上から順番にブロックごとに独立した実験が行われています。

**主な確認ポイント:**
*   **ゼロ値 (Zero Value)**: 変数を宣言しただけで値を入れなかった場合、勝手に入る初期値のこと（数字なら `0`、文字列なら `""` 空文字）。
*   **リテラル**: プログラム中に直接書かれた値（`10` や `"Hello"` など）の書き方。
    *   `1_234` のようにアンダースコアで桁区切りができる。
    *   `0x` (16進数)、`0b` (2進数) などの表記。
*   **宣言方法**:
    *   `var x int = 10` （丁寧な宣言）
    *   `x := 10` （省略記法。型推論が働く）
    *   `var x, y = 10, 20` （まとめて宣言）

**実行してみよう:**
```bash
go run ex0200.go
```
出力結果を見ながら、コードの対応する部分を読んでみてください。「なぜこう表示されるのか？」を考えるのが学習の近道です。

### 2. 複素数 (`ex0201.go`, `ex0201x.go`)

Go言語は科学技術計算も視野に入れているため、**複素数 (Complex Number)** を組み込み型として持っています。

*   `complex(実部, 虚部)` 関数で作るか、

日常的なWeb開発ではあまり使いませんが、Goの機能の広さを知る良い例です。

### 3. 型変換 (`ex0202.go`, `ex0203.go`)

**Go言語は型にとても厳しい言語です。**
例えば、「整数 (`int`)」と「浮動小数点数 (`float64`)」は別の型なので、そのままでは足し算すらできません。

**悪い例（コンパイルエラー）:**
```go
var x int = 10
var y float64 = 30.2
var sum = x + y // エラー！ 型が違うものを計算できない
```

**良い例（型変換を使う）:**
計算する前に、どちらかの型に合わせる必要があります。これをキャスト（型変換）と呼びます。

```go
// ex0202.go より
var sum1 float64 = float64(x) + y // xをfloat64に変換してから足す
var sum2 int = x + int(y)         // yをintに変換（小数点以下切り捨て）してから足す
```

`ex0203.go` では `byte` 型（0〜255までの整数）との変換を行っています。

### 4. 定数のルール (`ex0204.go`)

定数 (`const`) は、「一度決めたら変わらない値」です。
このファイルは**エラーが出ることを確認するためのコード**です。

```go
// ex0204.go より
const x int64 = 10

func main() {
	x = x + 1 // コンパイルエラー！ 定数は変更できない
}
```

**実行してエラーを確認しよう:**
```bash
go run ex0204.go
```
以下のようなエラーが表示されれば正解です。
`cannot assign to x` （xに代入できません）

### 5. ユニークな変数名 (`ex0206.go`, `ex0207.go`)

Go言語では、変数名に Unicode（ユニコード）文字を使うことができます。つまり、日本語や絵文字も変数名になります。

**ex0206.go の例:**
```go
func main() {
	_0 := 0_0
	_𝟙 := 20      // 特殊な数字文字
	π := 3        // ギリシャ文字のパイ
	ａ := "hello" // 全角のａ
	数値１ := 33   // 日本語
    // ...
}
```

**ex0207.go の注意点:**
全角の `ａ` と半角の `a` は**全く別の変数**として扱われます。
バグの原因になりやすいので、特別な理由がない限りは**英数字（半角）のみ**を使うのがプログラミングの定石です。
